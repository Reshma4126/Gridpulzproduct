# GridPulz Project File Documentation

This document describes the purpose of the files and folders in the workspace.

## Root Files

- `.env` — Local environment configuration loaded by the backend and launcher script. Typically contains Supabase and other secret values.
- `run_backend.bat` — Starts the backend server. It activates `.venv` and launches FastAPI with Uvicorn on `http://127.0.0.1:8000`.
- `setup_backend_env.bat` — Creates the Python virtual environment and installs backend dependencies from `backend/requirements.txt`.

## Backend

- `backend/main.py` — FastAPI application entrypoint. It configures CORS, loads environment variables, connects to Supabase, loads the EV prediction model, and exposes the API endpoints for station status, station registration, and emergency reroute actions.
- `backend/requirements.txt` — Python dependency list for the backend service.
- `backend/__pycache__/` — Generated Python bytecode cache. Safe to ignore.

## Frontend Pages

- `frontend/index.html` — Public landing page and homepage. Contains the main marketing hero, product overview, and links into login and informational pages.
- `frontend/about.html` — About page describing the product mission, vision, and company story.
- `frontend/features.html` — Features page that presents the main product capabilities and navigation back to the login flow.
- `frontend/contact.html` — Contact / sales page for reaching the team.
- `frontend/login.html` — Authentication gateway for operator and user login.
- `frontend/signup-operator.html` — Operator registration page used to create and configure charging station accounts.
- `frontend/signup-user.html` — EV driver registration page for user profile creation.
- `frontend/dashboard.html` — Main operator dashboard. Shows live station metrics, load data, predictions, and navigation to alerts and systems.
- `frontend/alerts.html` — Critical alerts page for overload and emergency handling workflows.
- `frontend/systems.html` — Systems monitoring page for hardware, connectivity, firmware, and infrastructure health.

## Frontend Scripts

- `frontend/js/auth.js` — Handles login and signup flows for operators and users, including Supabase auth and station registration submission.
- `frontend/js/operator.js` — Drives the operator dashboard. Fetches station data, updates live metrics, simulates dashboard values when needed, and redirects to alerts on overload.
- `frontend/js/alerts.js` — Powers the alert experience, including the alarm sound, acknowledgement flow, and emergency reroute sequence.
- `frontend/js/supabase-config.js` — Initializes the global Supabase client used by the frontend.

## Frontend Styles

- `frontend/css/styles.css` — Shared stylesheet for the frontend. Contains global typography overrides, shared UI helpers, and reusable dashboard / utility styles.

## Frontend Assets

- `frontend/assets/gridpulz_logo.png` — Main GridPulz brand logo.
- `frontend/assets/gridpulz_hero.png` — Hero graphic used in marketing or landing-page sections.
- `frontend/assets/ev_hero_bg.png` — Background image for EV-themed hero sections.
- `frontend/assets/ev_car_hero.png` — EV car illustration used in promotional content.

## Model Asset

- `frontend/ml model/ev_model.pkl` — Serialized machine-learning model used by the backend for load prediction when available.

## Generated / Local-Only Folders

- `.venv/` — Local Python virtual environment. Generated by `setup_backend_env.bat` and not meant to be edited directly.
- `backend/__pycache__/` — Python cache output generated at runtime.

## Notes

- The frontend uses Space Grotesk as the shared font across pages.
- The backend expects Supabase credentials in `.env` or environment variables.
- The live dashboard and systems pages rely on the FastAPI backend for station data when the operator is authenticated.
